# GitHub Scout 網頁步驟

## 用小學生的語言解釋

想像你在玩一個尋寶遊戲：

### 步驟 1: 在輸入框打字 📝
輸入框就像一個郵箱，你在那裡寫下你要找的人的名字。你打字輸入一個GitHub用戶名，比如"alice"或"bob"。

*背後發生什麼:* 你的電腦讀取你打的字，並把它存起來像一張小紙條。

### 步驟 2: 按下按鈕 🔘
當你按下按鈕，就像在GitHub的家門按門鈴，問"嘿！你認識叫'alice'的人嗎？可以告訴我關於他們的事嗎？"

*發生什麼:* 你的網頁發送一個訊息到GitHub的電腦（叫做"伺服器"），請他們搜尋那個人。

### 步驟 3: 照片出現 📸
GitHub翻翻他們的檔案，找到"alice"，然後回傳資訊，包括他們的**個人頭像**。照片神奇地出現在你的網頁上！

*發生什麼:* 
1. GitHub找到那個人
2. 他們把照片回傳給你的網頁
3. 你的網頁把它顯示在螢幕上

### 如果你按按鈕但沒有打字怎麼辦？ ❌
哎呀！如果你忘記在輸入框裡打字就按了按鈕會發生什麼？

網頁會說"等等！你忘記告訴我你要找誰！"。它會停止，什麼都不會發生。就像你去郵局但沒有寫人的名字在信封上——郵局的人不知道該去哪裡找！

*背後發生什麼:* 你的網頁聰明到知道要檢查輸入框是不是空的。如果是空的，它就說"不行，先打一個名字吧！" 📛

---

**整個過程就像：**
- 📮 你**寫一封信**（在輸入框打字）
- 🚪 你**按門鈴**（按按鈕）
- 📬 **郵件到達**帶著照片（照片出現）
